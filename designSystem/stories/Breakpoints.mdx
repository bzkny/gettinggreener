### Usage
Breakpoints are a mixin allowing @include to be dropped in as-needed into components. See the example below for a tablet specific mixin for a h1:

```
    h1 {
    @include respond-to-(tablet) {
        //ALSO .display-m
        font-size: 6.75rem;
    }
}
```

##

### Accessibility

Breakpoints should be written in em for accessibility. It is the only unit that scale's according to a user's prescribed font-size (as set in their browser settings) and zoom settings so is optimal for vision impaired users who need larger font sizes. See: <a href="https://medium.com/zoosk-engineering/to-em-or-not-to-em-that-is-the-media-query-question-22f4a65e9747" target="_blank" title="2019 research demonstrates em is most accessible unit for breakpoints">2019 research</a> and <a href="https://zellwk.com/blog/media-query-units/" target="_blank" title="2016 research demonstrates em is most accessible unit for breakpoints"> 2016 research</a>.

N.B. This em breakpoint will necessitate em/rem being used for padding, margins, font-sizes, and widths (where applicable).
